# 项目工作协议

本项目是一个以本地文件系统为主库的长期学习知识库，围绕 `02_library/`、`03_vault/`、Zotero 和 Codex 协同维护。

- 目录只表达稳定维度：学科、分支、资料类型。
- 不要随意新增顶层目录；先复用现有结构。
- 新资料必须先进入 `01_inbox/`，完成去重、重命名、分类后再入库。
- 新增学科或分支前，必须先更新 `00_system/taxonomy/subjects.yaml`。
- 新笔记优先使用 `00_system/templates/` 或 `03_vault/08_templates/` 中的模板。
- 不要随意改已有实体资料文件名；只在 inbox 标准化阶段重命名。
- 涉及结构变更时，优先修改规则文档与脚本，再修改具体内容。
- 生成或修改内容前，先检查是否会覆盖已有非空文件。
- 输出时先总结变更，再列目录，再给下一步建议。
