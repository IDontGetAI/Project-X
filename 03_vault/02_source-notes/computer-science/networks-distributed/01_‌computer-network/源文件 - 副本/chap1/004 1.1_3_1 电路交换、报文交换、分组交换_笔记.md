##### 1) 电路交换 ﻿01:44﻿

- 电路交换技术通过电话交换机实现节点通信，早期依赖人工接线，后期实现电气化与自动化。 
- 核心原理：动态分配物理线路资源，建立专用连接供通信双方独占使用。 
- 电路交换的过程 ﻿03:06﻿
- 建立连接：通信双方通过交换机分配物理线路资源（如拨号接通）。 
- 通信阶段：双方独占线路资源直至通信结束。 
- 释放连接：挂断后归还资源。 
- 资源占用差异：长途通信需占用更多线路资源，导致成本更高。 
- 线路冲突：若目标线路被占用，新请求将提示“占线”。 
- 电路交换的优点 ﻿06:28﻿
- 传输速率高：专用线路确保数据直达，延迟低。 
- 稳定性强：连接建立后通信质量稳定。 
- 电路交换的缺点 ﻿07:01﻿
- 连接开销大：建立 / 释放连接需额外时间，高频次短通信效率低。 
- 资源利用率低：线路独占导致无法共享。 
- 灵活性差：故障需重建连接，无法动态调整路径。 
- 无差错控制：中间节点不校验数据，长距离传输易出错。 
- 电路交换更适用场景 ﻿07:42﻿
- 低频次大量数据传输（如长时间语音通话），不适用于计算机网络的突发式通信需求。 
- 报文交换技术（电报网络） 
- 报文组成：控制信息（源 / 目的地址） $+$ 用户数据。 
- 存储转发机制： 
  - 交换机接收报文并暂存。 
  - 解析控制信息，查询转发表选择路径。 
  - 根据网络负载动态选择空闲线路转发。 
- 优势：支持多路径选择，资源利用率高于电路交换。

##### 2) 如果计算机网络采用电路交换技术 ﻿15:20﻿

报文交换技术在通信前无需建立连接，仅需确保每个报文包含必要的控制信息（如源地址、目的地址）和用户数据。报文长度可变： 

- 文字聊天报文可能仅含少量字节数据 
- 视频文件报文可能达到数 $GB$ 级别 

报文交换工作流程： 

- 报文从源计算机发送至交换机，交换机需存储报文 
- 交换机分析控制信息后决定转发路径 
- 同一通信线路可被多对用户共享，线路利用率显著提高 

报文交换核心优势： 

- 存储转发机制支持灵活路径分配，故障时可动态调整路由 
- 中间节点可实施差错控制技术（如奇偶校验），提升长距离传输可靠性 

报文交换主要缺陷： 

- 不定长报文导致管理困难： 
  - 大报文需占用大量交换机内存资源 
  - 内存分配算法复杂度增加（对比固定分区管理） 
- 长报文处理效率低： 
  - 校验时间随报文长度线性增长 
  - 传输错误概率与报文长度正相关，错误重传代价高昂 

#### 3.分组交换技术 ﻿23:18﻿

分组交换技术将长报文切分为定长分组，每个分组包含： 

- 首部信息（含源地址、目的地址、分组序号） 
- 数据块（固定长度） 

关键设计原理： 

- 分组序号确保接收方能按序重组原始数据 
- 首部信息（Header）控制分组传输路径，与报文交换控制功能等效 

##### 1) 例题:分组交换技术示例 ﻿24:57﻿

分组传输示例流程： 

- 源数据被拆分为分组 $0/1/2$，经不同路径传输 
- 动态路由选择：分组 $2$ 因上行链路拥堵改道下行链路 
- 接收方按分组序号重组数据，去除首部后还原用户数据 

现代网络实现： 

- 路由器作为分组交换机，通过 IP 地址判断转发路径 
- 分组定长特性简化存储管理，降低处理延迟 

##### 2) 分组交换的优缺点 ﻿27:15﻿

| 对比维度 | 报文交换             | 分组交换               |
| -------- | -------------------- | ---------------------- |
| 数据单位 | 不定长报文           | 定长分组               |
| 存储管理 | 内存需求不可控       | 固定块管理更高效       |
| 差错控制 | 长报文校验开销大     | 短分组校验效率显著提升 |
| 重传代价 | 整报文重传资源消耗高 | 单分组重传代价低       |

分组交换特有缺陷： 

- 首部开销增加：控制信息占比高于报文交换 
- 存储转发时延：各节点需处理首部信息，速度低于电路交换 
- 传输可靠性问题：可能出现分组失序或分组丢失，需额外处理机制

#### 4.虚电路交换技术 ﻿30:45﻿

虚电路交换技术的本质是分组交换。在通信双方传输数据之前，需要先建立连接，再传送数据，最后释放连接。

##### 1) 虚电路交换的过程 ﻿31:01﻿

虚电路交换的工作流程如下：

- 数据分组：发送方将数据拆分为多个分组
- 连接建立：发送端向接收端发送建立虚拟连接的请求
- 路径选择：网络建立一条虚拟电路作为传输路径
- 数据传输：分组按序通过虚拟电路传输
- 资源共享：传输链路可被其他用户共享
- 数据重组：接收端按序接收分组并重组为完整数据

技术特点：

- 本质：分组交换技术
- 创新：吸收了电路交换的连接建立思想
- 现代应用：未被现代计算机网络采用

未被采用的原因：

- 终端性能：现代计算机处理能力强，可解决分组失序等问题
- 网络效率：将排序责任交给网络核心会降低传输效率
- 设计理念：充分发挥终端计算能力更符合现代网络架构
